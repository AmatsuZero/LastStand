# 面试准备练习清单

## 阶段一：稿子写熟（每天1小时，3天）

- [ ] 自我介绍脱稿练习 5 遍，卡在 2 分钟以内
- [ ] 3D 球场三段式脱稿练习 3 遍，确保追问能自然回答
- [ ] PC 开播助手三段式脱稿练习 3 遍
- [ ] AIRead SDK 三段式脱稿练习 3 遍
- [ ] 管理短板回答脱稿练习 5 遍（这是最容易被问到的）

## 阶段二：模拟问答（每天1小时，2天）

- [ ] 找个朋友模拟面试，完整走一遍流程（30分钟）
- [ ] 让朋友随机追问，练习即兴回答
- [ ] 录下来回听，找卡壳和啰嗦的地方
- [ ] 高频问题随机抽 5 个，练习 1 分钟内给出完整回答

## 阶段三：临场准备（面试前1天）

- [ ] 把面试稿打印出来，面试前快速过一遍
- [ ] 准备好博客链接（GitHub Pages 地址），面试中适时展示
- [ ] 准备好反问问题的优先级排序
- [ ] 确认面试时间、形式（线上/线下）、面试官背景

## 面试当天

- [ ] 提前 10 分钟到位，调整状态
- [ ] 自我介绍开场（用练习的稿子）
- [ ] 项目深挖时先给三段式总览，再根据面试官兴趣展开
- [ ] 被问管理短板时用三段论回答，不回避不心虚
- [ ] 反问环节问 2-3 个问题，展现 Owner 意识
- [ ] 结束后复盘：哪些问题回答得好/不好，记录下来

## 回答质量标准

好的回答特征：
- 有结构（三段式），不是想到哪说到哪
- 有具体例子，不是空谈理论
- 有决策过程（为什么选A不选B），不是只讲结果
- 控制在 1-2 分钟，不是长篇大论

差的回答特征：
- "我们团队做了..."（应该是"我做了..."）
- 技术术语堆砌，没有业务价值
- 回避问题，答非所问
- 太谦虚，不敢展示自己的贡献
